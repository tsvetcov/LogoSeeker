
## ML-Пайплайн

Модерация выполняется в два этапа для разделения зон ответственности моделей:
* **Детекция (YOLOv8)** локализует любые логотипы на комплексном фоне, вырезает их координаты и передает фрагменты на следующий уровень. 
* **Классификация (DINOv2)** извлекает эмбеддинги из вырезанных логотипов и рассчитывает косинусное сходство с эталонными массивами векторов.

### Бизнес-логика вердиктов:
* `ОДОБРЕНО (ok)` — логотип принадлежит разрешенной категории (собственные бренды) либо не найден в базах нарушителей (неизвестный бренд).
* `НА РУЧНУЮ МОДЕРАЦИЮ` — обнаружено частичное сходство с конкурентом или запрещенным брендом (попадание в серую зону порогов уверенности `0.40 - 0.85`).
* `ЗАБЛОКИРОВАНО` — жесткое совпадение с запрещенным брендом или конкурентом (уверенность `> 0.85`).

## Требования для запуска

Для запуска проекта на локальной машине необходимы:
* [Docker Desktop](https://www.docker.com/products/docker-desktop/)
* Установленный плагин `docker-compose` (входит в состав Docker Desktop)


## Подготовка к первому запуску

Поскольку веса нейросетей и векторные базы данных имеют большой объем, они **исключены из Git-репозитория** через `.gitignore`. Перед сборкой контейнеров выполните следующие шаги:

### 1. Клонирование репозитория DINOv2
Для корректной инициализации экстрактора векторов необходимо склонировать официальную кодовую базу DINOv2 в папку классификатора:
```bash
cd classifier
git clone [https://github.com/facebookresearch/dinov2.git](https://github.com/facebookresearch/dinov2.git)
cd ..
````

### 2. Размещение весов моделей

1. Скачайте обученные веса детектора `best.pt` (YOLO) и предобученные веса классификатора `dinov2`.
2. Поместите файл `best.pt` в папку, указанную в конфигурации пайплайна (по умолчанию корень).

### 3. Размещение векторных баз эталонов

Убедитесь, что в папке `bases/` присутствуют соответствующие файлы `.npy` (векторные массивы, сгенерированные в ходе исследований в Jupyter-ноутбуках), которые будут использоваться для расчета косинусного сходства.

## Инструкция по запуску

Все команды выполняются из корневой директории проекта (где находится файл `docker-compose.yml`).

### 1. Сборка и запуск контейнеров в фоновом режиме:

Bash

```
docker-compose up --build -d
```

### 2. Остановка сервисов:

Для остановки контейнеров без потери данных в БД:

Bash

```
docker-compose down
```

Если вам необходимо полностью очистить кэш базы данных и запустить СУБД с чистого листа, используйте флаг удаления разделов:

Bash

```
docker-compose down -v
```

## Доступ к сервисам

После успешного развертывания сервисы доступны по следующим адресам:

- **Интерфейс модерации (Frontend):** [http://localhost](https://www.google.com/search?q=http://localhost&authuser=1) (Порт 80)
- **Документация бэкенда и ручное тестирование API (Swagger):** [http://localhost:8000/docs](https://www.google.com/search?q=http://localhost:8000/docs&authuser=1)
- **Порт СУБД (MariaDB):** `3307` (внутренний порт контейнера `3306` проброшен наружу как `3307` во избежание конфликтов с локально установленными СУБД).
